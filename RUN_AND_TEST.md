# 포토모자이크 웹앱 실행/테스트 가이드

이 문서는 이 저장소의 웹앱을 **실행하고 테스트하는 방법**을 한 곳에 정리한 문서입니다.

## 1) 요구 환경

- Node.js 18 이상 권장
- npm 9 이상 권장
- 최신 브라우저(Chrome, Edge, Safari, Firefox)

확인 명령어:

```bash
node -v
npm -v
```

---

## 2) 설치 및 실행

저장소 루트(`/workspace/Mosaic`)에서 아래 순서로 실행합니다.

```bash
npm install
npm run dev
```

성공 시 Vite 개발 서버 주소가 출력됩니다. (예: `http://localhost:5173`)
브라우저에서 해당 주소를 열어 앱을 사용합니다.

---

## 3) 프로덕션 빌드 및 미리보기

```bash
npm run build
npm run preview
```

- `npm run build`: 배포용 정적 파일 생성
- `npm run preview`: 빌드 결과 로컬 확인

---

## 4) 기능 테스트 체크리스트 (수동)

아래 시나리오를 순서대로 점검하면 핵심 기능을 검증할 수 있습니다.

### A. 대표 이미지 업로드
1. 대표 이미지 파일(jpg/jpeg/png) 1장을 선택한다.
2. 미리보기 영역에 이미지가 즉시 보이는지 확인한다.
3. 다른 대표 이미지로 교체했을 때 미리보기가 갱신되는지 확인한다.

### B. 타일 이미지 업로드/관리
1. 타일 이미지 여러 장을 한 번에 선택한다.
2. 썸네일 그리드에 이미지가 추가되는지 확인한다.
3. 개별 `삭제` 버튼이 동작하는지 확인한다.
4. `전체 삭제` 버튼이 동작하는지 확인한다.

### C. 설정 패널
1. 타일 크기(8/12/16/20/24/32) 변경이 가능한지 확인한다.
2. 반복 사용 허용 토글이 동작하는지 확인한다.
3. 출력 가로 크기(800/1200/1600) 변경이 가능한지 확인한다.
4. 출력 비율(원본/1:1/4:5/16:9) 변경이 가능한지 확인한다.
5. 미리보기 품질(low/medium/high) 변경이 가능한지 확인한다.

### D. 생성 버튼 활성화 조건
1. 대표 이미지가 없으면 `모자이크 생성` 버튼이 비활성화인지 확인한다.
2. 타일 이미지가 없으면 `모자이크 생성` 버튼이 비활성화인지 확인한다.
3. 대표+타일 모두 있으면 버튼이 활성화되는지 확인한다.

### E. 모자이크 생성 및 진행 상태
1. `모자이크 생성` 클릭 시 진행 메시지가 바뀌는지 확인한다.
2. 진행률/처리 셀 수가 표시되는지 확인한다.
3. 생성 완료 후 결과 캔버스가 출력되는지 확인한다.

### F. 결과/다운로드
1. 원본 이미지와 결과 미리보기가 함께 보이는지 확인한다.
2. 생성 정보(원본 크기/타일 수/셀 개수/출력 크기/타일 크기)가 보이는지 확인한다.
3. `PNG 다운로드` 클릭 시 `photo-mosaic.png` 파일이 저장되는지 확인한다.

### G. 초기화
1. `전체 초기화` 클릭 시 대표 이미지/타일/결과/진행 상태가 초기화되는지 확인한다.

---

## 5) 예외/오류 테스트

1. 이미지가 아닌 파일 업로드 시 안내 문구가 뜨는지 확인한다.
2. 타일이 매우 적을 때 품질 경고 문구가 보이는지 확인한다.
3. 생성 중 오류 발생 시 사용자 친화적 오류 문구가 표시되는지 확인한다.

---

## 6) 참고: 현재 환경에서 발생 가능한 설치 이슈

일부 제한된 실행 환경에서는 `npm install` 시 레지스트리 접근 제한으로 `403 Forbidden`이 발생할 수 있습니다.
이 경우:

1. 네트워크/방화벽/프록시 정책 확인
2. npm registry 설정 확인 (`npm config get registry`)
3. 가능한 환경(로컬 PC 등)에서 동일 명령 재시도
